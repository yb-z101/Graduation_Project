# 系统部署文档框架

## 一、部署概述

### 1.1 系统架构
本系统采用前后端分离的B/S架构：
- **前端**：Vue 3 + Element Plus + ECharts
- **后端**：FastAPI + Python + Pandas + LangGraph
- **数据库**：MySQL 8.0+

### 1.2 部署环境要求

#### 1.2.1 硬件要求
- **最低配置**：
  - CPU：2核
  - 内存：4GB
  - 硬盘：20GB可用空间
- **推荐配置**：
  - CPU：4核及以上
  - 内存：8GB及以上
  - 硬盘：50GB可用空间

#### 1.2.2 软件要求
- **操作系统**：Windows 10/11、Linux（Ubuntu 20.04+）、macOS 10.15+
- **Python**：3.10 或更高版本
- **Node.js**：16 或更高版本
- **MySQL**：8.0 或更高版本
- **Web服务器**：Nginx（生产环境推荐）

## 二、环境准备

### 2.1 安装Python

#### Windows
1. 下载Python 3.10+：https://www.python.org/downloads/
2. 运行安装程序，勾选"Add Python to PATH"
3. 验证安装：
   ```bash
   python --version
   pip --version
   ```

#### Linux (Ubuntu)
```bash
sudo apt update
sudo apt install python3.10 python3.10-venv python3-pip
python3 --version
pip3 --version
```

### 2.2 安装Node.js

#### Windows
1. 下载Node.js 16+：https://nodejs.org/
2. 运行安装程序
3. 验证安装：
   ```bash
   node --version
   npm --version
   ```

#### Linux (Ubuntu)
```bash
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt install -y nodejs
node --version
npm --version
```

### 2.3 安装MySQL

#### Windows
1. 下载MySQL 8.0+：https://dev.mysql.com/downloads/mysql/
2. 运行安装程序，设置root密码
3. 启动MySQL服务

#### Linux (Ubuntu)
```bash
sudo apt update
sudo apt install mysql-server
sudo mysql_secure_installation
sudo systemctl start mysql
sudo systemctl enable mysql
```

#### 创建数据库
```sql
CREATE DATABASE graduation_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'graduation_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON graduation_db.* TO 'graduation_user'@'localhost';
FLUSH PRIVILEGES;
```

## 三、后端部署

### 3.1 克隆或上传代码
```bash
cd /path/to/deploy
# 或上传项目文件到服务器
```

### 3.2 创建虚拟环境

#### Windows
```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
```

#### Linux/Mac
```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
```

### 3.3 安装依赖
```bash
pip install -r requirements.txt
```

### 3.4 配置环境变量
复制 `.env` 示例文件并编辑：
```bash
cp .env.example .env
# 或编辑现有的 .env 文件
```

配置内容：
```env
# ==================== MySQL 数据库配置 ====================
MYSQL_USER=graduation_user
MYSQL_PASSWORD=your_password
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DB=graduation_db

# ==================== 大模型 API 配置 ====================
# 火山引擎 API 密钥
VOLCENGINE_API_KEY=your_volcengine_api_key

# 星火大模型配置
SPARK_APP_ID=your_spark_app_id
SPARK_API_KEY=your_spark_api_key
SPARK_API_SECRET=your_spark_api_secret

# 通义千问 API 密钥
QWEN_API_KEY=your_qwen_api_key

# 深度求索 API 密钥
DEEPSEEK_API_KEY=your_deepseek_api_key

# ==================== 应用配置 ====================
# CORS 允许的域名（生产环境修改为实际域名）
CORS_ORIGINS=["https://your-domain.com"]

# 应用名称
APP_NAME=毕业项目智能分析系统

# 应用版本
APP_VERSION=1.0.0
```

### 3.5 启动后端服务

#### 开发环境
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

#### 生产环境（使用Gunicorn）
```bash
# 安装Gunicorn
pip install gunicorn uvloop httptools

# 启动服务
gunicorn -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000 main:app
```

#### 使用Systemd管理（Linux）
创建服务文件 `/etc/systemd/system/chat-analysis-backend.service`：
```ini
[Unit]
Description=Chat Analysis Backend
After=network.target

[Service]
Type=notify
User=www-data
WorkingDirectory=/path/to/backend
Environment="PATH=/path/to/backend/.venv/bin"
ExecStart=/path/to/backend/.venv/bin/gunicorn -w 4 -k uvicorn.workers.UvicornWorker -b 127.0.0.1:8000 main:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start chat-analysis-backend
sudo systemctl enable chat-analysis-backend
```

### 3.6 验证后端部署
访问 http://your-server-ip:8000/docs 查看API文档。

## 四、前端部署

### 4.1 安装依赖
```bash
cd frontend
npm install
```

### 4.2 配置环境变量

#### 开发环境
编辑 `.env.development`：
```env
VUE_APP_API_BASE_URL=http://localhost:8000/api/v1
```

#### 生产环境
编辑 `.env.production`：
```env
VUE_APP_API_BASE_URL=https://your-domain.com/api/v1
```

### 4.3 构建生产版本
```bash
npm run build
```

构建完成后，会在 `frontend/dist` 目录生成静态文件。

### 4.4 部署到Web服务器（Nginx）

#### 安装Nginx
```bash
# Ubuntu/Debian
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx
```

#### 配置Nginx
创建配置文件 `/etc/nginx/sites-available/chat-analysis`：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 增加上传文件大小限制
    client_max_body_size 20M;
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/chat-analysis /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4.5 配置HTTPS（推荐）
使用Let's Encrypt免费证书：
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 五、Docker部署（可选）

### 5.1 后端Dockerfile
创建 `backend/Dockerfile`：
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 5.2 前端Dockerfile
创建 `frontend/Dockerfile`：
```dockerfile
FROM node:16-alpine as build

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 5.3 Docker Compose
创建 `docker-compose.yml`：
```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: graduation_db
      MYSQL_USER: graduation_user
      MYSQL_PASSWORD: user_password
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  backend:
    build: ./backend
    environment:
      MYSQL_HOST: mysql
      MYSQL_USER: graduation_user
      MYSQL_PASSWORD: user_password
      MYSQL_DB: graduation_db
      # 其他环境变量...
    depends_on:
      - mysql
    ports:
      - "8000:8000"

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
```

启动服务：
```bash
docker-compose up -d
```

cat > /path/to/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -u graduation_user -p'your_password' graduation_db > /path/to/backups/db_$DATE.sql
find /path/to/backups -name "db_*.sql" -mtime +7 -delete
EOF

chmod +x /path/to/backup.sh

# 添加到crontab，每天凌晨2点备份

## 七、故障排查

### 7.1 后端无法启动
- 检查Python版本是否符合要求
- 检查虚拟环境是否激活
- 检查依赖是否完整安装
- 检查数据库连接配置
- 查看后端日志：`backend/logs/`

### 7.2 前端无法访问
- 检查Nginx是否运行
- 检查防火墙设置
- 检查前端构建是否成功
- 查看浏览器控制台错误

### 7.3 大模型API调用失败
- 检查API密钥配置
- 检查网络连接
- 确认账户余额充足
- 查看后端日志中的错误信息

### 7.4 文件上传失败
- 检查文件大小是否超限
- 检查Nginx的client_max_body_size配置
- 检查磁盘空间是否充足
- 检查文件权限

## 八、安全建议

1. **修改默认密码**：所有系统账户使用强密码
2. **配置防火墙**：只开放必要的端口（80、443）
3. **启用HTTPS**：使用SSL/TLS加密传输
4. **定期更新**：及时更新系统和依赖包
5. **日志审计**：定期检查访问日志
6. **限制API访问**：配置API限流和认证
7. **数据加密**：敏感数据加密存储
8. **备份验证**：定期验证备份的可用性
